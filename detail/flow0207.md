# AMS360 Automation — Multi-Window Orchestrator Flow (v3)

## Overview

This document describes the flow after `flow2.md`. The desktop side no longer
assumes every action leads to the Policy window — it now dispatches to the
correct desktop window (Policy / Renew / Endorse / Receipt) based on which
action button was clicked in the browser grid, and each window's session
class only implements what that window actually needs.

---

## What Changed vs `flow2.md`

1. **Desktop orchestrator now dispatches by action** instead of always
   opening the Policy window.
   - **File:** `desktop/orchestrator.py`
   - New `DesktopOrchestrator.run_flow(action, customer, fee, instruction)` replaces the
     hardcoded call to `run_policy_flow()` in the workflow.
   - Routes to `RenewWindowSession`, `EndorseWindowSession`, `ReceiptWindowSession`,
     or falls back to the existing `run_policy_flow()` for `"Policy"` / unrecognized actions.
   - `run_policy_flow()` itself is unchanged — it's just no longer the only path.

2. **`PolicyInstruction.action` widened.**
   - **File:** `excel/models.py`
   - `Literal["Renew", "Endorse"]` → `Literal["Renew", "Endorse", "Policy", "Receipt"]`
   - Note: the browser side (`PolicyService.perform_grid_action` / `PolicyGrid.perform_action`)
     still only knows how to click Renew/Endorse toolbar buttons, so `"Policy"`/`"Receipt"`
     aren't reachable from the grid-click flow yet — only from a future flow that opens
     those windows a different way.

3. **Window title matching fixed for Renew.**
   - **File:** `config/constants.py`
   - `RENEW_WINDOW` was a literal string (`"Create Renewal/Rewrite Policy - Policy number"`)
     that never matched the real window title, which ends in the actual policy number
     (e.g. `Create Renewal/Rewrite Policy - ACI04B11019405`). It's now a partial regex
     so it matches regardless of which policy number is in the title.

4. **Renew window now reads Effective/Expiration Date via Tab navigation.**
   - **File:** `desktop/renew_window/reader.py`, `desktop/renew_window/session.py`
   - Both the Effective Date and Expiration Date edit controls share the same
     `auto_id` (`ZZZm_txtDateValue`), so they can't be told apart by `child_window()`
     lookup alone. Instead, the session presses `Tab` to move focus field-by-field
     and reads whichever control currently has focus via UI Automation's
     `GetFocusedElement`.

5. **Policy grid rows scoped to the correct grid via its header.**
   - **File:** `config/constants.py`, `browser/policy_grid.py`
   - `POLICY_GRID_ROW` was a bare `div.x-grid3-row` selector, which would also match
     rows from any other ExtJS grid on the page.
   - It's now scoped to `div.x-grid3-viewport:has(.x-grid3-hd-Policy) div.x-grid3-row`
     — i.e. only rows inside the viewport whose header row contains the `Policy` column
     are fetched.

6. **Receipt window placeholder added.**
   - **File:** `desktop/receipt_window/session.py` (new)
   - `ReceiptWindowSession` is a stub (logs and returns) — controls haven't been
     inspected yet. It exists purely so `run_flow()` has a branch to call once the
     real window is available.

## Each Window Flow

### Policy Window Flow

**Entry point:** `DesktopOrchestrator.run_policy_flow(customer, fee)` — unchanged
from `flow2.md`.

1. `DesktopManager.wait_window(WINDOW_TITLES.POLICY_WINDOW)` → `attach()` → `maximize()`
2. `PolicyWindowSession.run(fee)`:
   - `BasicInfo.open()` — expand Basic Policy Information, set Status
     (`Active` for Annual term, `Non-renewed` otherwise)
   - `FeeHandler.add(charge_type, amount)` — demo fee entry, cancelled (not saved)
   - `PolicyReader.read()` — parses policy number/dates from the window title,
     reads Premium / Fees & Taxes / Annualized Premium / Bill Method controls
   - Returns `PolicyInfo`
3. `customer.name` / `customer.address` stamped onto the result
4. `DesktopManager.close(window)` (waits `WINDOW_CLOSE_WAIT` seconds first)

Returns `PolicyInfo`, which the workflow writes to Excel.

---

### Renew Window Flow

**Entry point:** `DesktopOrchestrator._run_renew_flow(instruction)`, dispatched from
`run_flow()` when `instruction.action == "Renew"`.

1. `DesktopManager.wait_window(WINDOW_TITLES.RENEW_WINDOW)` → `attach()` → `maximize()`
   (title matched via the partial regex `"Create Renewal/Rewrite Policy.*"`)
2. `RenewWindowSession.run(instruction.policy_number)`:
   - `RenewWindowReader.get_policy_number()` — reads current Policy # (`txtCopyPolicy_..._BPolPolNo`)
   - `press_tab()` → `get_focused_field_value()` — tabs onto the Effective Date field
     and logs its value
   - `press_tab()` → `get_focused_field_value()` — tabs onto the Expiration Date field
     and logs its value
   - `set_policy_number(instruction.policy_number)` — overwrites the Policy # field
   - `click_cancel()`
3. `DesktopManager.close(window)`

Returns `None` — this flow doesn't produce a `PolicyInfo`, so nothing is written
to Excel for it (yet).

---

### Endorse Window Flow

**Entry point:** `DesktopOrchestrator._run_endorse_flow(instruction)`, dispatched from
`run_flow()` when `instruction.action == "Endorse"`.

1. `DesktopManager.wait_window(WINDOW_TITLES.ENDORSE_WINDOW)` → `attach()` → `maximize()`
2. `EndorseWindowSession.run(instruction.endorsement_description)`:
   - `EndorseWindowReader.set_description(description)`
3. `DesktopManager.close(window)`

Returns `None`.

---

### Receipt Window Flow (stub — not yet implemented)

**Entry point:** `DesktopOrchestrator._run_receipt_flow()`, dispatched from
`run_flow()` when `instruction.action == "Receipt"`.

1. `DesktopManager.wait_window(WINDOW_TITLES.RECEIPT_WINDOW)` → `attach()` → `maximize()`
   (title constant `"Receipt"` is a placeholder — not confirmed against the real window)
2. `ReceiptWindowSession.run()` — currently just logs a warning; no controls implemented
3. `DesktopManager.close(window)`

Nothing else happens yet. This branch exists so the dispatcher is ready once the
Receipt window's controls are inspected.

## Updated Dispatch Logic

**File:** `desktop/orchestrator.py` → `DesktopOrchestrator.run_flow()`

```python
if action == "Renew":
    self._run_renew_flow(instruction)
elif action == "Endorse":
    self._run_endorse_flow(instruction)
elif action == "Receipt":
    self._run_receipt_flow()
else:
    return self.run_policy_flow(customer=customer, fee=fee)
```

Called from `workflow/ams360_workflow.py`:

```python
policy_info = self.desktop.run_flow(
    action=instruction.action, customer=customer, fee=fee, instruction=instruction
)
if policy_info is not None:
    self.excel_service.write_policy(policy_info)
```

## Pending / Not Yet Implemented

- **Receipt window controls** — title, field auto_ids, and session logic all need
  inspection against the real AMS360 Receipt window.
- **Browser-side dispatch for Policy/Receipt actions** — `PolicyGrid.perform_action()`
  still only supports `"Renew"`/`"Endorse"` button clicks; there's no grid action yet
  that would produce `instruction.action == "Policy"` or `"Receipt"`.
- **Renew/Endorse window outputs** — neither flow currently returns data to write to
  Excel; only the Policy window flow produces a `PolicyInfo`.
- **Excel Input Loop** — carried over from `flow2.md`; still reads from `.env`, not
  yet looping over rows in an input Excel file.
